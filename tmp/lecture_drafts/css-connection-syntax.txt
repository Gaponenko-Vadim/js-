# CSS: подключение и базовый синтаксис

## Введение: как инструкции на упаковке 📦

Представьте две одинаковые коробки: одна с понятной инструкцией, другая — без неё.  
В первом случае всё собирается быстро, во втором — хаос.

CSS — это инструкция для внешнего вида страницы.  
HTML говорит «что», а CSS объясняет «как это выглядит».

💡 **Совет:** начинайте с простого и аккуратного CSS — это экономит часы правок.  
✅ **Вывод:** правильное подключение и чистый синтаксис — основа красивой страницы.

---

## Проблема: стили не применяются ❌

**Без понимания подключения:**
- CSS «не работает»;
- стили конфликтуют;
- правки превращаются в случайность.

**С пониманием подключения:**
- стили применяются предсказуемо;
- код читается и поддерживается;
- дизайн масштабируется.

✅ **Вывод:** 80% проблем с CSS — это неверное подключение и порядок правил.

---

## Ключевые термины (простыми словами)

- **CSS (Cascading Style Sheets, каскадные таблицы стилей)** — язык оформления HTML.  
- **Selector (селектор)** — «кого» мы оформляем.  
- **Declaration (декларация)** — «что» меняем: `property: value;`.  
- **Rule (правило)** — селектор + блок деклараций.  
- **Cascade (каскад)** — порядок применения стилей.  
- **Specificity (специфичность)** — «важность» селектора.  
- **External stylesheet (внешний файл)** — отдельный `.css`.  
- **Inline style (инлайн‑стиль)** — стиль прямо в `style="..."`.  

✅ **Вывод:** эти термины часто спрашивают на собеседованиях.

---

## 1. Подключение CSS через `<link>` (внешний файл)

**Назначение:** хранить стили отдельно от HTML.  
**Аналогия:** инструкция в отдельной книжке.

**Пример:**
~~~html
<link rel="stylesheet" href="styles.css" />
~~~

**Характеристики:**
- ✅ лучший вариант для реальных проектов;
- ✅ один файл можно подключать к разным страницам.

**Когда использовать:** почти всегда.
✅ **Вывод:** внешний CSS — стандарт индустрии.

---

## 2. Подключение через `<style>` (внутренний стиль)

**Назначение:** быстро оформить одну страницу.  
**Аналогия:** заметки на полях.

**Пример:**
~~~html
<style>
  body { font-family: Arial, sans-serif; }
</style>
~~~

**Характеристики:**
- ✅ удобно для прототипов;
- ❌ тяжело поддерживать в больших проектах.

**Когда использовать:** учебные задания, быстрые черновики.
✅ **Вывод:** удобно, но не масштабируется.

---

## 3. Inline‑стили (`style="..."`)

**Назначение:** точечная правка элемента.  
**Аналогия:** наклейка поверх инструкции.

**Пример:**
~~~html
<button style="background: #111; color: #fff;">Купить</button>
~~~

**Характеристики:**
- ❌ плохо переиспользуется;
- ❌ делает код грязным;
- ✅ иногда полезен для быстрых тестов.

**Когда использовать:** редко, только для единичных случаев.
✅ **Вывод:** inline — плохая привычка в проектах.

---

## 4. `@import` внутри CSS

**Назначение:** подключить другой CSS‑файл из файла стилей.  
**Аналогия:** «вставить главу» из другой книги.

**Пример:**
~~~css
@import "reset.css";
~~~

**Характеристики:**
- ✅ удобно для разбивки файлов;
- ❌ может замедлять загрузку по сравнению с `<link>`.

**Когда использовать:** для вспомогательных файлов, если нет сборки.
✅ **Вывод:** лучше `<link>`, но `@import` иногда полезен.

---

## 5. Базовый синтаксис CSS

**Назначение:** описать стиль для выбранного элемента.

**Пример:**
~~~css
h1 {
  color: #111;
  font-size: 32px;
}
~~~

**Характеристики:**
- ✅ правило = селектор + декларации;
- ✅ каждая декларация заканчивается `;`.

**Когда использовать:** всегда при написании стилей.
✅ **Вывод:** чистый синтаксис = предсказуемый результат.

---

## 6. Селекторы: тег, класс, id

**Назначение:** указать, к чему применяется стиль.

**Пример:**
~~~css
h2 { color: #111; }
.card { border: 1px solid #e5e7eb; }
#main { max-width: 960px; }
~~~

**Характеристики:**
- ✅ **tag** — все элементы;
- ✅ **class** — переиспользуемые стили;
- ✅ **id** — уникальный элемент.

**Когда использовать:** class — почти всегда, id — только для уникальных.
✅ **Вывод:** классы — основа масштабируемого CSS.

---

## 7. Каскад и порядок правил

**Назначение:** понять, какой стиль победит.

**Пример:**
~~~css
p { color: #111; }
.card p { color: #4b5563; }
~~~

**Характеристики:**
- ✅ более конкретный селектор выигрывает;
- ✅ правило, которое ниже, может перекрыть верхнее.

**Когда использовать:** при конфликте стилей.
✅ **Вывод:** каскад — основная «магия» CSS.

---

## 8. Специфичность (очень кратко)

**Пример:**
~~~css
p { color: #111; }          /* низкая */
.card p { color: #333; }    /* выше */
#main p { color: #000; }    /* ещё выше */
~~~

✅ **Вывод:** чем точнее селектор, тем выше приоритет.

---

## Сравнение способов подключения

| Способ | Плюсы | Минусы | Когда использовать |
|---|---|---|---|
| `<link>` | масштабируется, чистый код | нужен отдельный файл | почти всегда |
| `<style>` | быстро, удобно для прототипов | сложно поддерживать | учебные, черновики |
| inline | мгновенно, локально | грязно, не переиспользуется | очень редко |
| `@import` | дробит стили | может быть медленнее | вспомогательные стили |

---

## Часто спрашивают на собеседованиях

- **Чем отличается `<link>` от `<style>`?** Внешний файл vs стили внутри HTML.
- **Когда применять inline‑стили?** Только в исключительных случаях.
- **Что такое каскад?** Порядок применения правил.
- **Как работает специфичность?** Чем точнее селектор — тем выше приоритет.
- **Зачем нужен класс, если есть id?** Класс переиспользуется, id — уникален.

✅ **Вывод:** эти вопросы — база для junior‑интервью.

---

## Типичные ошибки

### Ошибка 1: CSS‑файл не подключён
❌ Неправильно: нет `<link>`.
✅ Правильно: `<link rel="stylesheet" href="styles.css">`.

### Ошибка 2: Стили в `<style>` ниже `<link>`
❌ Неправильно: порядок случайный.
✅ Правильно: общий порядок понятен и контролируем.

### Ошибка 3: Inline‑стили повсюду
❌ Неправильно: сложно поддерживать.
✅ Правильно: стили в классах.

### Ошибка 4: Неправильный селектор
❌ Неправильно: `card {}` вместо `.card {}`.
✅ Правильно: точка для класса.

### Ошибка 5: Пропущена `;`
❌ Неправильно: `color: #111`.
✅ Правильно: `color: #111;`.

### Ошибка 6: Конфликт правил
❌ Неправильно: непонятно, что победит.
✅ Правильно: контроль каскада и специфичности.

---

## Best Practices

- Подключайте CSS через `<link>`.
- Держите стили в одном‑двух файлах на старте.
- Используйте классы, а не inline‑стили.
- Пишите понятные селекторы.
- Следите за порядком правил и специфичностью.
- Делайте короткие, читаемые блоки.

---

## Популярные сценарии

### Типичные ошибки (сводка)
- Файл CSS не подключён.
- Inline‑стили вместо классов.
- Неправильный селектор (`card` вместо `.card`).
- Стили конфликтуют из‑за каскада.
- Пропущены `;` в правилах.

---

### Сценарий 1: Стили не применяются

❌ Неправильно: нет подключения CSS‑файла.  
✅ Правильно: `<link>` в `<head>`.

**До:**
~~~html
<head>
  <title>Курс</title>
</head>
~~~

**После:**
~~~html
<head>
  <title>Курс</title>
  <link rel="stylesheet" href="styles.css" />
</head>
~~~

**Почему**: браузер не знает, откуда брать стили.

**Практический пример:**
~~~
Все страницы должны подключать styles.css.
~~~

---

### Сценарий 2: Стили разбросаны inline

❌ Неправильно: много `style="..."`.  
✅ Правильно: класс + правило в CSS.

**До:**
~~~html
<button style="background:#111;color:#fff;">Записаться</button>
~~~

**После:**
~~~html
<button class="btn-primary">Записаться</button>
~~~

~~~css
.btn-primary { background: #111; color: #fff; }
~~~

**Почему**: так проще поддерживать и переиспользовать.

**Практический пример:**
~~~
Кнопки должны оформляться через классы.
~~~

---

### Сценарий 3: Неправильный селектор класса

❌ Неправильно: `card {}`.  
✅ Правильно: `.card {}`.

**До:**
~~~css
card { border: 1px solid #e5e7eb; }
~~~

**После:**
~~~css
.card { border: 1px solid #e5e7eb; }
~~~

**Почему**: класс обозначается точкой.

**Практический пример:**
~~~
Все карточки должны иметь рамку.
~~~

---

### Сценарий 4: Конфликт правил

❌ Неправильно: правило выше перекрывается, но это не учли.  
✅ Правильно: используем более конкретный селектор.

**До:**
~~~css
p { color: #111; }
~~~

**После:**
~~~css
p { color: #111; }
.card p { color: #4b5563; }
~~~

**Почему**: более конкретный селектор выигрывает.

**Практический пример:**
~~~
Текст в карточках должен быть серым.
~~~

---

### Сценарий 5: Пропущена `;`

❌ Неправильно: правило ломается.  
✅ Правильно: каждая декларация завершается точкой с запятой.

**До:**
~~~css
h1 { color: #111 font-size: 32px }
~~~

**После:**
~~~css
h1 { color: #111; font-size: 32px; }
~~~

**Почему**: CSS перестаёт читать правило корректно.

**Практический пример:**
~~~
Все свойства заканчиваются `;`.
~~~

---

### Сценарий 6: `<style>` ниже `<link>`

❌ Неправильно: случайный порядок.  
✅ Правильно: осознанный порядок стилей.

**До:**
~~~html
<link rel="stylesheet" href="styles.css" />
<style>h1 { color: red; }</style>
~~~

**После:**
~~~html
<style>h1 { color: red; }</style>
<link rel="stylesheet" href="styles.css" />
~~~

**Почему**: последняя подключенная запись может перекрыть предыдущую.

**Практический пример:**
~~~
Приоритет нужно контролировать через порядок.
~~~

## Задания для практики 🧪

### Задание 1: Подключить внешний CSS
**Кратко:**
Ситуация: стили не применяются.
Что сделать: подключить styles.css через `<link>`.
Критерий: в `<head>` есть `<link rel="stylesheet">`.

**Описание:**
✅ **Что сделать:** добавьте `<link rel="stylesheet" href="styles.css" />` в `<head>`.
🎯 **Критерий:** браузер видит внешний файл стилей.

🧱 **Исходник (как сейчас):**
~~~html

~~~

**Ответ:**
**Было:**
~~~html

~~~

**Стало:**
~~~html
<link rel="stylesheet" href="styles.css" />
~~~

**Объяснение:** без `<link>` внешний CSS не подключается.
