# Инструкция по созданию заданий (TASKS)

Кратко: задания — это практическая часть лекции. Они должны помогать студенту **делать один проект шаг за шагом**, закрепляя ключевые концепции темы.

---

## 1. Назначение заданий

- Закрепить знания из лекции через практику.
- Дать ощущение «я сделал реальный результат».
- Сформировать навык, а не просто проверить память.

**Главный принцип:** все задания одной лекции дополняют **один общий проект**.

---

## 2. Где живут задания

Задания хранятся в поле `tasksContent` у лекции и отображаются во вкладке **"Задания"**.
Формат строго Markdown с понятными блоками (ниже).

---

## 3. Общая структура `tasksContent`

```
## Подготовка к заданиям
[цель + контекст + как работать]

[каркас проекта]

## Задания для практики 🧪

### Задание 1: ...
**Кратко:**
Ситуация: ...
Что сделать: ...
Критерий: ...

**Описание:**
[детальные требования, контекст, подсказки]

🧱 **Исходник (как сейчас):**
[код/документ]

**Ответ:**
**Было:**
[код/документ]

**Стало:**
[код/документ]

**Объяснение:** 2–4 предложения
```

---

## 4. Блок «Подготовка к заданиям» (обязателен)

**Зачем:** это инструкция и каркас проекта. Студент вставляет каркас в редактор и дальше **дорабатывает его по заданиям**.

**Что должно быть:**
- 🎯 **Цель заданий** (зачем эти упражнения).
- 🧩 **Контекст проекта** (что за продукт/документ/код).
- ✅ **Как работать** ("вставьте каркас и дополняйте по заданиям").
- **Каркас** (готовый стартовый шаблон).

**Важно:** каркас должен быть **полным**, с контейнером и корректной структурой. Не только `<main>`, а полноценный каркас для работы.

### Каркас для разных тем

**HTML/CSS:**
- Полноценная HTML-страница (doctype, html, head, body, контейнеры).
- Если используются классы — добавьте inline‑стили, чтобы не было «пустого» вида.

**Документация/API:**
- Каркас из секций документа: Цель, Требования, Критерии, Примеры.
- В каждом задании дополняется часть этого документа.

**Общее правило:** каркас должен быть **универсальным**, чтобы студент мог вставить его в любой редактор и сразу увидеть результат.

---

## 5. Как связаны задания между собой

- Задания **идут цепочкой**.
- Каждое следующее задание **дополняет** результат предыдущего.
- Нельзя делать «разрозненные» задания, которые не связаны общим проектом.

✅ Пример логики:
1. Список преимуществ
2. Шаги процесса
3. Термины
4. Иерархия
5. Таблица сравнения
6. Структура таблицы
7. Доступность
8. Итог

---

## 6. Формат задания

### 6.1. Заголовок
```
### Задание N: Короткое название
```
Название должно быть коротким и понятным.

### 6.2. Блок «Кратко» (видно на карточке)
Формат всегда одинаковый:
```
**Кратко:**
Ситуация: ...
Что сделать: ...
Критерий: ...
```
**Важно:** карточка — это только короткое резюме. Никаких деталей и подсказок.

### 6.3. Блок «Описание» (полное задание)
- Контекст и ситуация.
- Чёткие требования.
- Упоминание обязательных атрибутов (id, aria, scope и т.д.).
- Что получится в итоге.

### 6.4. Исходник
Всегда даём **как сейчас** в код‑блоке. Это видно по кнопке «Задание».

### 6.5. Ответ + объяснение
- Ответ показывается **по кнопке "Ответ"**.
- Формат: **Было** → **Стало** → **Объяснение**.
- Объяснение — 2–4 предложения: почему так, что улучшилось, как проверить.

---

## 7. Контентные правила

- Задания должны охватывать **все ключевые концепции лекции**.
- В каждом задании должна быть **понятная цель** и **критерий проверки**.
- Если используется специальный атрибут/термин — он должен быть описан в требованиях.
- В "Кратко" нельзя раскрывать ответ.

---

## 8. Оформление

- Используйте понятные эмодзи (🎯, ✅, 🧱), но без перегруза.
- Кодовые блоки короткие, читаемые.
- Форматирование всегда единообразное.

---

## 9. Чек‑лист качества

- [ ] Есть блок **Подготовка к заданиям** с каркасом проекта.
- [ ] Каркас полный и понятный, не пустой.
- [ ] Все задания связаны в один проект.
- [ ] 7–15 заданий (по сложности темы).
- [ ] У каждого задания есть «Кратко», «Описание», «Исходник», «Ответ», «Объяснение».
- [ ] В «Кратко» нет деталей решения.
- [ ] Ответ объясняет **почему**, а не только показывает код.
- [ ] В заданиях указаны все обязательные атрибуты.
- [ ] Задания покрывают ключевые концепции лекции.
